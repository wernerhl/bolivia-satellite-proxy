# Fires, Lights, and Smog — Data, Figures, and Tables Agent Brief

## Your scope

**You produce the inputs. Claude writes the paper.**

Your job is to acquire data, run empirical estimates, and generate the figures and table fragments that the paper v3 skeleton references. You do not write or modify paper prose. You do not fill in the `\pend{...}` placeholder text in the LaTeX body. You do not touch `fires_lights_smog_v3.tex`. If you think a result deserves a sentence in the paper, that is a note to Claude, not an edit by you.

Inputs arrive in three buckets:

- **Data files** to `/data/processed/*.parquet` — one file per source, schema specified below.
- **Figure PDFs** to `/figures/pdf/fig{1..5}_*.pdf` and `/figures/pdf/figA{1,2}_*.pdf` — seven files, exact filenames.
- **Table fragments** to `/tables/table{1..4}_*.tex` and `/tables/tableA{1,2}_*.tex` — six files, exact filenames, each a self-contained `\begin{table}...\end{table}` block.

That is the full deliverable surface. The paper compiles from these plus the v3 .tex.

## Rule zero: no placeholder figures

The v2 run produced figure files with "TBD: stream not yet available" text rendered inside them and autogenerated DFM fit statistics reported in prose even though three of the four streams were missing. This was worse than not producing the figures at all, because the output looked like a result and a casual reader could not tell it was not one.

The new rule: **if any input required for a figure or a table cell is not available, that figure is not produced and that cell reads `---`**.

Specifically:

- If VNF Chaco monthly data does not exist, Figure 1 panel 2 is blank or missing entirely. Not "TBD stream not yet available" text.
- If the DFM does not converge, Figure 3 is not produced. Not "DFM not yet fit" rendered text.
- If YPFB field-month production is not available, Figure A2 is not produced. Not six panels of "YPFB field-month required" text.
- If a table cell requires an estimate that has not been produced, the cell contains exactly `---`. No "inputs_missing", no "insufficient_n", no numerical-looking placeholders like `0.00` or `$n=---$` inside estimate columns.

When a figure cannot be produced, write a one-line note to `/figures/pending.log` stating which figure is missing and which input is required. The paper compiles around missing PDFs because `graphicx` errors on missing files — to avoid that, generate a blank placeholder PDF with a single line of text reading `Figure pending: requires <input name>`, no axes, no fake plot.

When a table cell cannot be filled, use `---`. Do not generate placeholder numbers. Do not generate fake confidence intervals. Do not fit models to partial data and report the output.

## Rule one: nothing leaves your hands that is fake

- No invented numbers in captions.
- No auto-filled `\claim{...}` paragraphs in the paper body. That path was how "the satellite factor bottoms at $z=-2.62$ in 2024-01" ended up in the abstract of the v2 draft despite the underlying series not yet being fetched. Never write to the main `.tex` file. Not even in comments, not even inside `\iffalse...\fi`, not even as a note. Your edits are confined to `/data/`, `/figures/`, and `/tables/`.
- If you must communicate a finding to Claude, write it to `/notes/for_claude.md`. Claude will read it and decide whether it goes in the paper.

## Rule two: date boundaries are hard

- VIIRS DNB starts 2012-04 (when `VNP46A3` becomes routinely available).
- VIIRS Nightfire starts 2012-01.
- TROPOMI NO$_2$ starts 2018-07.
- Sentinel-2 L2A starts 2017-03. Pre-2017 NDVI uses Landsat-8 (`LANDSAT/LC08/C02/T1_L2`, operational 2013-04) with Roy et al. (2016) cross-sensor harmonization.

Under no circumstances does a time-series plot x-axis begin before 2012. The v2 draft's Figure 3 ran from 1992 because a time index defaulted to century start. That bug invalidates the figure. Every figure script begins with `ax.set_xlim(pd.Timestamp("2012-04-01"), pd.Timestamp("2026-05-01"))` or equivalent.

## Rule three: deterministic and reproducible

Every artifact regenerates from a single `make` target reading from a frozen Parquet dataset. Random seeds are fixed. Figure layouts are not hand-tuned. Overwriting an output with the same input must produce a byte-identical file (modulo PDF creation timestamps, which are fine to differ).

## Data tracks (produce in order)

### D1 — INE quarterly GDP, sectoral, 2017 base

- **Source.** INE Nextcloud portal, Excel files under Cuentas Nacionales, 2017-base series.
- **Coverage required.** 2012Q1 through the latest release (2025Q4 as of April 2026).
- **Fields.** Quarter, sector (10 sectors per INE classification plus total), nominal value, real value (2017 base), y/y growth rate, q/q growth rate.
- **Output.** `/data/processed/ine_gdp_quarterly.parquet` with one row per (quarter, sector). Total GDP as its own sector code `TOTAL`.
- **Manual step.** No API. Download XLSX manually and archive to `/data/raw/ine/YYYY-MM-DD/` with ISO-dated folder. The scraper script, when run, reads from the most recent raw folder.

### D2 — INE departmental GDP, annual

- **Source.** INE annual Cuentas Departamentales publication, XLSX.
- **Coverage required.** 2013 through 2024 (latest available). Annual.
- **Fields.** Year, department (9 departments), sector (10 sectors), nominal value, real value (2017 base where available; earlier vintages are in the 1990 base and must be chain-spliced at the 2013 overlap).
- **Output.** `/data/processed/ine_dep_gdp_annual.parquet`. Chain-splice explicitly, document the splice factor in a separate column `splice_factor`.

### D3 — INE experimental monthly IGAE

- **Source.** INE portal, bulletin PDF plus XLSX annex.
- **Coverage required.** March 2026 onward (that is the entire series at submission vintage — roughly 2 months of data).
- **Fields.** Month, IGAE index (2017 = 100), year-on-year growth, month-on-month growth, trend-cycle, seasonally-adjusted series.
- **Output.** `/data/processed/ine_igae_monthly.parquet`. Flag every row as `preliminary = True` until INE declassifies the series.

### D4 — YPFB field-month gas production

- **Source.** YPFB Boletín Estadístico de Hidrocarburos, quarterly publication with monthly field breakout as Excel annex.
- **Coverage required.** January 2012 through latest available month.
- **Fields.** Year, month, field name (7 fields), operator, gas production (MMm³/day monthly average), condensate production (bbl/day), well count.
- **Field name mapping** (match to VNF extraction in D8 exactly):
  - Margarita → `margarita`
  - Huacaya → `huacaya`
  - San Alberto → `san_alberto`
  - Sábalo → `sabalo` (note: ASCII transliteration)
  - Incahuasi → `incahuasi`
  - Aquio → `aquio`
  - Itaú → `itau`
- **Output.** `/data/processed/ypfb_field_monthly.parquet`. One row per (year, month, field).
- **Source note.** PDFs live at `ypfb.gob.bo/boletin-estadistico/`. Use `pdfplumber` on the field-annex table. Archive raw PDFs under `/data/raw/ypfb/YYYY-MM-DD/`.

### D5 — IBCH cement dispatches monthly

- **Source.** Instituto Boliviano del Cemento y el Hormigón, monthly Despachos de Cemento bulletin. Members-only publication; Fundación Milenio republishes the national and departmental aggregates.
- **Coverage required.** January 2012 through latest month.
- **Fields.** Year, month, department (or national), dispatches (tonnes).
- **Output.** `/data/processed/ibch_cement_monthly.parquet`.

### D6 — CNDC electricity generation and demand

- **Source.** Comité Nacional de Despacho de Carga, daily dispatch dashboard with XLSX download.
- **Coverage required.** Daily 2012-01-01 through latest available, aggregated to monthly for the DFM.
- **Fields.** Date, generation (MWh), demand (MWh), peak demand (MW), reserve margin (%), source split (thermal, hydro, renewable).
- **Output.** `/data/processed/cndc_electricity_daily.parquet` at daily frequency; `/data/processed/cndc_electricity_monthly.parquet` as the month-sum for DFM use.

### D7 — SIN tax collections monthly, real

- **Source.** Servicio de Impuestos Nacionales, monthly Recaudación bulletin.
- **Coverage required.** January 2012 through latest month.
- **Fields.** Year, month, tax type (IVA, IT, IUE, ICE, others), nominal collection (Bs), real collection (Bs, 2018 = 100, deflated by INE CPI).
- **Output.** `/data/processed/sin_tax_monthly.parquet`.

### D8 — VIIRS Day/Night Band sum-of-lights

- **Source.** Google Earth Engine, `NOAA/VIIRS/001/VNP46A3` (monthly Black Marble product).
- **Coverage required.** April 2012 through the most recent month with a two-month publication lag (so as of April 2026, February 2026 is the latest).
- **Method.**
  1. Load the collection, filter to date range.
  2. For each of the 11 urban buffers in Table 1 of the paper (see `/config/spatial.yaml`), extract the mean `AllAngle_Composite_Snow_Free` radiance within the circular buffer.
  3. Mask pixels with radiance `< 0.5` nW·cm⁻²·sr⁻¹ (background) or `> 200` nW·cm⁻²·sr⁻¹ (aberration).
  4. Compute sum-of-lights (SOL) as the sum of valid radiance over the buffer.
  5. Report mean, median, valid-pixel count, masked-pixel count.
- **Fields.** Date (month-start), buffer name, SOL, mean, median, n_valid_pixels, n_masked.
- **Output.** `/data/processed/viirs_dnb_monthly.parquet`.
- **Deseasonalization.** Compute STL decomposition per buffer (monthly period, robust=True) on the 2013-01 to 2019-12 baseline window. Store the seasonal+trend components and the residual as separate columns.

### D9 — VIIRS Nightfire Chaco

- **Source.** EOG VNF archive at `eogdata.mines.edu/products/vnf/`. Not on GEE; requires EOG Earthdata Bearer token stored in `EOG_TOKEN` environment variable.
- **Coverage required.** January 2012 through latest month.
- **Method.**
  1. For each night, fetch detections within the Chaco bounding box `lat ∈ [-22.5, -20.0], lon ∈ [-64.5, -62.5]`.
  2. Filter: `Cloud_Mask == 0` AND `Temp_BB >= 1400 K` (flare regime; excludes biomass burning).
  3. Assign each detection to the nearest of the 7 fields (field centroids in `/config/spatial.yaml`) within a 2 km radius. Drop unassigned detections.
  4. Aggregate to field-month: sum of `RH` (radiant heat, MW), mean of `Temp_BB`, count of detections, count of cloud-blocked nights.
  5. Report Chaco aggregate as the sum across all 7 fields.
- **Fields.** Date (month-start), field name, rh_mw_sum, n_detections, mean_temp_bb, n_cloud_blocked_nights.
- **Output.** `/data/processed/vnf_chaco_monthly.parquet` with both field-level rows and a `TOTAL_CHACO` aggregate row per month.
- **Note.** Do not substitute World Bank GGFR annual flaring data for this stream. WB-GGFR is derived from VNF but aggregated to annual and globally calibrated; the paper's Test 1 requires raw monthly VNF at the field level.

### D10 — Sentinel-5P TROPOMI NO₂

- **Source.** Google Earth Engine, `COPERNICUS/S5P/OFFL/L3_NO2`.
- **Coverage required.** July 2018 through latest month (typically 10-day lag).
- **Method.**
  1. For each of the 3 ROIs (La Paz--El Alto, Santa Cruz, Cochabamba — see `/config/spatial.yaml`), extract the mean `tropospheric_NO2_column_number_density` with QA filter `qa_value >= 0.75`.
  2. Daily extraction, then monthly mean with `n_valid_days >= 15` required to report.
  3. Flag months with Andean volcanic contamination (`mean > 5 × baseline`) for manual review.
- **Fields.** Date (month-start), ROI, no2_tropos_col, n_valid_days, sd_daily, contamination_flag.
- **Output.** `/data/processed/s5p_no2_monthly.parquet`.
- **Baseline.** Month-of-year seasonal factor computed on 2019 only (single clean year pre-COVID, pre-Bolivia-crisis). Document that seasonal factors are point estimates with no SE until additional baseline years accumulate.

### D11 — Sentinel-2 NDVI over Bolivian cropland

- **Source.** Google Earth Engine, `COPERNICUS/S2_SR_HARMONIZED` from 2017-03 forward; `LANDSAT/LC08/C02/T1_L2` from 2013-04 to 2017-02 for the spliced baseline.
- **Coverage required.** January 2013 through latest month.
- **Method.**
  1. For each of the 5 cropland zones, load the image collection.
  2. Cloud mask via SCL band (exclude SCL classes 3, 8, 9, 10, 11 for Sentinel-2; QA_PIXEL for Landsat-8).
  3. Compute NDVI = (B8 - B4) / (B8 + B4) for Sentinel-2; NDVI = (SR_B5 - SR_B4) / (SR_B5 + SR_B4) for Landsat-8.
  4. Apply ESA WorldCover 2021 cropland mask (class 40) to keep only cropland pixels.
  5. Compute monthly median NDVI per zone over crop-masked pixels.
  6. Apply Roy et al. (2016) harmonization to Landsat-8 NDVI before splicing: `NDVI_S2 = 1.00 * NDVI_L8 + 0.01`.
- **Fields.** Date, zone, ndvi_median, n_valid_pixels, sensor ("S2" or "L8_harmonized").
- **Output.** `/data/processed/s2_ndvi_monthly.parquet`.

### D12 — Aduana imports monthly

- **Source.** INE Nextcloud xlsx (INE republishes Aduana data with a cleaning layer).
- **Coverage required.** January 2012 through latest month.
- **Fields.** Year, month, import category (capital, intermediate, consumption, fuel), USD CIF, volume (tonnes).
- **Output.** `/data/processed/aduana_imports_monthly.parquet`.

### D13 — BCB monetary execution

- **Source.** Banco Central de Bolivia, monthly Seguimiento al Programa Monetario PDF.
- **Coverage required.** January 2012 through latest month.
- **Fields.** Month, emisión monetaria, CIN total, CIN-SPNF, CIN-Sector Financiero, OMAs, RIN (USD).
- **Output.** `/data/processed/bcb_monetary_monthly.parquet`.

### D14 — External forecasts

- **Source.** IMF WEO April 2026 vintage, World Bank GEP January 2026 vintage, Oxford Economics Country Economic Forecast latest, S&P Sovereign Risk Indicator report latest.
- **Coverage required.** Annual forecasts for 2025, 2026, 2027.
- **Fields.** Year, forecaster, forecast_real_gdp_growth_pct, vintage_date.
- **Output.** `/data/processed/external_forecasts_annual.parquet`.

## Analysis tracks

### E1 — Single-series elasticities

After D1, D2, D4, D8, D9, D10, D11, D12 complete:

- Estimate equation (1) (VIIRS → GDP, departmental panel): regress $\Delta \log(\text{GDP}_{d,t})$ on $\Delta \log(\text{SOL}_{d,t})$ with department fixed effects and year fixed effects. Two-way clustered SE by department and year.
- Estimate equation (2) (VNF → gas production, field panel): same specification, field fixed effects and time fixed effects. Pre-crisis 2012–2022 calibration window only.
- Estimate equation (3) (NO₂ → fuel sales, metro panel): with post-December-2025 fuel-subsidy break indicator.
- Estimate equation (4) (NDVI → agricultural GVA, departmental panel).

Output: `/data/estimates/elasticities.parquet` with one row per specification, columns `beta, se, t, pvalue, n, r2, specification`.

### E2 — Two-factor DFM

After D1, D5, D6, D7, D8, D9, D10, D11, D12 complete:

- Assemble the 9-variable monthly panel: VIIRS DNB composite, TROPOMI NO$_2$ composite, IBCH cement, CNDC electricity, SIN real IVA, Aduana imports (urban block); VNF Chaco aggregate, YPFB gas production, hydrocarbon-sector electricity (extractive block). All standardized to z-scores on 2013–2019.
- Estimate the two-factor DFM per equations (4)–(6) of the paper using `statsmodels` or `dynamic_factor_mq`. EM algorithm with Kalman smoothing.
- Apply Mariano–Murasawa aggregation (equation 7) to recover a quarterly-GDP-consistent monthly factor path.
- Compute BBQ turning points on the GDP-weighted composite.
- Estimate Hamilton 2-state Markov-switching AR(2) on the composite; report filtered recession probabilities.
- If EM fails to converge or factor loadings do not separate (evidence of single common component), fall back to Lewis–Mertens–Stock pre-specified weighted composite.

Output: `/data/estimates/dfm_factors_monthly.parquet` (filtered factors, recession probabilities); `/data/estimates/dfm_loadings.parquet` (factor loadings by series); `/data/estimates/bbq_turning_points.parquet`.

### E3 — Manipulation tests

After D1, D3, D4, D8, D9, D14 and E2 complete:

- **Test 1 (sectoral triangulation).** Pre-crisis 2012–2022 log-log regression of monthly VNF ΣRH against YPFB gas production (Chaco aggregate). Report pre-crisis β, SE, R². Compute 2023–2026 residuals against the extrapolated pre-crisis fit. Separately regress INE hydrocarbon VA on YPFB gas production, pre-crisis only, and compute 2023–2026 residuals.
- **Test 2 (leadership discontinuity).** Estimate equation (9) from the paper with controls. If the IGAE post-break sample has fewer than 10 observations, estimate on quarterly GDP with the MM aggregation, and flag as preliminary.
- **Test 3 (external-forecaster residual).** Annual satellite composite regressed separately on INE reported growth and on the mean of IMF/WB/Oxford/S&P forecasts. Report residual magnitudes and the t-test of equal magnitudes.

Output: `/data/estimates/manipulation_tests.parquet` with verdict column.

- No legend frame.

- Every figure script begins with explicit x-axis bounds on time plots (see Rule two).

### T1 — Spatial definitions

Pure metadata, no estimation required. Four sections: urban buffers (11 rows from `/config/spatial.yaml`), Chaco fields (7 rows), TROPOMI ROIs (3 rows), cropland zones (5 rows). Current v1 version is correct; copy through without modification.

### T2 — Elasticities

Input: E1 output. Four rows (VIIRS, VNF, NO₂, NDVI). Columns: Stream, Spec. equation, β̂ (SE), n, R², Range, Benchmark.

Rule: any cell whose value has not been estimated reads `---`. Do not generate placeholder numbers. Do not write "inputs_missing".

### T3 — Satellite vs external forecasters

Input: E2 composite, D1 INE, D14 forecasters. Four rows (2023, 2024, 2025, 2026). Columns: Year, INE (%), Sat. mean z, IMF (%), WB (%), Oxford (%), S&P (%).

If satellite composite is not yet estimated for a year, that cell reads `---`. If INE has not released the year, that cell reads `n/a`. Do not interpolate.

### T4 — Manipulation suite

Input: E3 output. Three rows (Test 1, Test 2, Test 3). Columns: Test, Identifying assumption, Statistic, Verdict.

Each verdict is one of: `Reject H₀ at 5%`, `Reject H₀ at 10%`, `Fail to reject`, `Preliminary (n<10)`, `---` (test not yet run).

No "inputs_missing" or "insufficient_n" strings. If a test was run and failed to reject, that is a real result and reads `Fail to reject`. If a test was not run, it reads `---`.

### TA1 — Vintages

Pure metadata. One row per data series from D1–D14 plus satellite streams. Columns: Series, Source, Frequency, Lag (days), Coverage in paper, Revision. Update "Coverage in paper" to the actual date range produced by the D-track for each series.

### TA2 — Robustness grid

Input: re-estimation of E2 under alternative specifications. Columns: Specification, Trough date, Trough σ, 2025 growth (%), 2026 growth (%).

Rule: do not run this table until the baseline DFM converges in E2. Each row requires its own re-estimation. Rows for which the alternative specification has not been run read `---` across all columns. Do not populate the baseline row with results and leave alternatives empty; that creates the illusion that the robustness exercise has been done when it has not.

## Handoff protocol

When a batch of artifacts is ready, write `/notes/handoff_YYYY-MM-DD.md` with:

1. Which D-tracks completed and which are blocked (with specific blockers — "YPFB Q4 2025 bulletin not yet published" is actionable; "data pending" is not).
2. Which E-tracks completed.
3. Which figures and tables regenerated cleanly.
4. Any unexpected findings worth flagging to Claude (e.g., "pre-crisis VNF–YPFB R² came in at 0.42; this suggests Test 1 should be reframed — recommend Claude read the revised E3 notes before updating §4.3").

Claude reads the handoff, decides whether any paper text needs to be written or revised, and updates `/fires_lights_smog_v3.tex` accordingly. You do not.

## Failure modes to avoid

- Auto-filling paper prose from pipeline output (v2's `% BEGIN-AUTO-headline` mechanism). Do not do this.
- Substituting a proxy data source without flagging (v2's WB-GGFR-for-VNF). Do not do this.
- Rendering placeholder figures that look like real figures with TBD text inside. Do not do this.
- Reporting fit statistics in captions or log files when the model was fit to partial or synthetic data. Do not do this.
- Populating "Baseline" rows of tables while leaving "Alternative" rows empty, which creates the false appearance that a full robustness exercise has been conducted.

## What counts as done

- All 14 D-tracks produce Parquet files that pass schema validation.
- All 3 E-tracks produce estimate Parquet files.
- All 5 main-text and 2 appendix figures exist at the specified paths and were generated from real data.
- All 6 table fragments compile when `\input{}`'d into the paper.
- `pdflatex -> bibtex -> pdflatex -> pdflatex` on `fires_lights_smog_v3.tex` produces a clean PDF.
- `/notes/for_claude.md` summarizes any surprises in the data that Claude should consider when filling the `\pend{}` placeholders in the paper.

At that point, and only at that point, Claude takes over to write results prose, fill the `\pend{}` blocks, and prepare the submission package.
